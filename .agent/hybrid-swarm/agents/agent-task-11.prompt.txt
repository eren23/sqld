# Task: Catalog system with system tables and statistics

Implement catalog as in-memory cache backed by heap tables. Bootstrap system tables on first run: sys_tables, sys_columns, sys_indexes, sys_constraints, sys_views, sys_statistics (all schemas as specified). Implement catalog operations: create/drop/alter table (updating both heap and cache transactionally), create/drop index, create/drop view, lookup by name/id. Implement ANALYZE command: scan table to compute TableStats (row_count, total_pages) and ColumnStats (null_count, distinct_count, min/max, avg_width, equi-depth histogram with 100 buckets, top-10 most common values with frequencies). Store statistics in sys_statistics. Implement VACUUM: mark dead tuples as reclaimable, update free space map, optional auto-vacuum (threshold + scale factor). Include tests for table/index/view CRUD, DROP CASCADE, concurrent DDL conflict, ANALYZE statistics accuracy (histogram coverage, MCV identification, distinct count), VACUUM space reclamation.

Target files: sqld/src/catalog/mod.rs, sqld/src/catalog/catalog.rs, sqld/src/catalog/system_tables.rs, sqld/src/catalog/statistics.rs, sqld/tests/test_catalog.rs, sqld/tests/test_statistics.rs, sqld/tests/test_vacuum.rs
