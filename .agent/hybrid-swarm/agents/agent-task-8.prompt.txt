# Task: Transaction manager with MVCC, locking, SSI, and savepoints

Implement Transaction struct: txn_id, status (Active/Committed/Aborted), isolation_level, snapshot (xmin/xmax/active_txns), write_set, read_set, savepoints, timestamps. Implement MVCC visibility rules: tuple visible if xmin committed+in-snapshot AND (xmax invalid OR xmax aborted OR xmax not-in-snapshot), with special handling for own-transaction writes. Implement Snapshot creation per isolation level: READ COMMITTED (new per statement), REPEATABLE READ (one per txn), SERIALIZABLE (one per txn + rw tracking). Implement lock manager: row-level Shared/Exclusive/SIRead locks, HashMap<LockTarget,LockEntry>, FIFO wait queue, intention locks for DDL. Implement deadlock detection: wait-for graph with DFS cycle detection every 100ms, abort victim with fewest writes. Implement SSI: SIREAD locks (advisory, never block), rw-dependency tracking, dangerous structure detection (T1→rw→T2→rw→T3 where T1,T3 concurrent), abort on commit. Implement savepoints: stack of (name, write_set_position, lock_count), ROLLBACK TO undoes writes after position, RELEASE discards marker. Include tests for all visibility rules, snapshot isolation, write-write conflicts, deadlock scenarios, SSI write-skew prevention, savepoint nesting.

Target files: sqld/src/transaction/mod.rs, sqld/src/transaction/transaction.rs, sqld/src/transaction/mvcc.rs, sqld/src/transaction/lock_manager.rs, sqld/src/transaction/ssi.rs, sqld/src/transaction/savepoint.rs, sqld/tests/test_mvcc_visibility.rs, sqld/tests/test_snapshot.rs, sqld/tests/test_transactions.rs, sqld/tests/test_isolation_read_committed.rs, sqld/tests/test_isolation_snapshot.rs, sqld/tests/test_isolation_serializable.rs, sqld/tests/test_lock_manager.rs, sqld/tests/test_deadlock.rs, sqld/tests/test_savepoints.rs
