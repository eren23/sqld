# Task: Write-ahead log, checkpointing, and ARIES recovery

Implement WAL record types: Begin/Commit/Abort, InsertTuple/DeleteTuple/UpdateTuple (physiological with full undo data), IndexInsert/IndexDelete, PageAlloc/PageFree, CheckpointBegin/CheckpointEnd, CLR (compensation log records). Each WalEntry has: lsn (byte offset), prev_lsn (per-txn undo chain), record, crc32. Implement WAL manager: 64KB in-memory buffer, flush on commit/buffer-full/10ms timeout, group commit (multiple txns share single fsync), monotonic LSN assignment. Implement checkpointing: write CheckpointBegin with active txns, flush all dirty pages, write CheckpointEnd, truncate WAL before checkpoint LSN. Implement ARIES 3-phase recovery: Analysis (scan from last checkpoint, rebuild dirty page table + active txn table), Redo (forward scan, re-apply if page LSN < record LSN, idempotent), Undo (walk undo chains for uncommitted txns, write CLRs, CLRs never undone). Include tests: record serialization round-trip, CRC integrity, LSN monotonicity, flush correctness, group commit, redo/undo recovery scenarios, checkpoint recovery, crash simulation at various points.

Target files: sqld/src/wal/mod.rs, sqld/src/wal/wal_record.rs, sqld/src/wal/wal_manager.rs, sqld/src/wal/recovery.rs, sqld/src/wal/checkpoint.rs, sqld/tests/test_wal_write.rs, sqld/tests/test_wal_flush.rs, sqld/tests/test_wal_protocol.rs, sqld/tests/test_recovery_redo.rs, sqld/tests/test_recovery_undo.rs, sqld/tests/test_recovery_checkpoint.rs, sqld/tests/test_recovery_nested.rs, sqld/tests/test_crash_simulation.rs
