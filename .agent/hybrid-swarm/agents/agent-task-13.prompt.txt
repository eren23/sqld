# Task: Query planner, cost-based optimizer, and physical plan selection

Implement logical plan: Scan/Filter/Project/Join/Aggregate/Sort/Limit/Distinct/Union/Intersect/Except/Insert/Update/Delete/Values/Empty nodes. Implement plan builder converting bound AST to logical plan tree following construction rules (FROM→Scan, WHERE→Filter, GROUP BY→Aggregate, etc.). Implement 10 logical rewrite rules: predicate pushdown, projection pushdown, constant folding, dead column elimination, subquery decorrelation (correlated→join), simplification (x AND true→x), IN-list to semi-join, EXISTS to semi-join, view merging, join elimination. Implement cardinality estimation using catalog statistics: selectivity formulas for equality/range/BETWEEN/LIKE/IS NULL/AND/OR/NOT/IN, join cardinality, default 0.1 for unknown predicates. Implement cost model with configurable constants for SeqScan/IndexScan/HashJoin/SortMergeJoin/NestedLoopJoin/HashAggregate/Sort. Implement join ordering: DP (DPccp) for ≤6 tables, greedy for 7-12, left-deep greedy for >12. Implement physical plan selection: choose among SeqScan/IndexScan, HashJoin/SortMergeJoin/NestedLoopJoin, HashAggregate/SortAggregate, ExternalSort, HashDistinct/SortDistinct. Implement EXPLAIN [ANALYZE] formatter. Include tests for logical plan construction, each optimization rule, cardinality estimates vs actuals, cost ranking, join reordering, index selection, EXPLAIN output.
