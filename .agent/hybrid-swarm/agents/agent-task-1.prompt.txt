# Task: Project scaffolding, type system, config, and error hierarchy

Initialize the Rust project with Cargo.toml (edition 2021, no external DB dependencies). Create lib.rs with all module declarations. Implement the types module: Datum enum (INTEGER/BIGINT/FLOAT/BOOLEAN/VARCHAR/TEXT/TIMESTAMP/DATE/DECIMAL/BLOB/NULL) with comparison, hashing, arithmetic, serialization, and coercion rules (INTEGER→BIGINT→DECIMAL→FLOAT, VARCHAR→TEXT, DATE→TIMESTAMP). Implement DataType enum with coercion lattice. Implement Schema (ordered column definitions, lookup by name/ordinal, merge for joins). Implement Tuple (row representation with null bitmap, fixed/variable-length fields, serialization/deserialization, MVCC header with xmin/xmax/cid). Implement config.rs to parse sqld_config.toml with all sections (server, storage, buffer_pool, wal, optimizer, transactions, vacuum, logging) with defaults. Implement utils/error.rs with a unified error hierarchy. Implement utils/pool.rs for reusable allocations and utils/metrics.rs for performance counters. Include unit tests for Datum comparison/coercion/hashing, Tuple round-trip serialization, Schema operations, and config parsing.

Target files: sqld/Cargo.toml, sqld/src/lib.rs, sqld/src/config.rs, sqld/src/types/mod.rs, sqld/src/types/datum.rs, sqld/src/types/data_type.rs, sqld/src/types/schema.rs, sqld/src/types/tuple.rs, sqld/src/utils/mod.rs, sqld/src/utils/error.rs, sqld/src/utils/pool.rs, sqld/src/utils/metrics.rs, sqld/tests/test_datum.rs, sqld/tests/test_tuple.rs, sqld/tests/test_schema.rs
