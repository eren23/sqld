# Task: SQL lexer and token definitions

Implement Token struct (kind, span byte-offset range, line, col) and TokenKind enum: all literals (IntegerLiteral/FloatLiteral/StringLiteral/BooleanLiteral/NullLiteral), identifiers (unquoted lowercased, quoted case-preserving), all SQL keywords (SELECT through ROW, 80+ keywords), operators (Plus/Minus/Star/Slash/Percent/Caret/Eq/NotEq/Lt/Gt/LtEq/GtEq/Concat), punctuation (parens, comma, semicolon, dot, ColonColon), Placeholder($N), Eof. Implement lexer: case-insensitive keyword matching, identifier rules [a-zA-Z_][a-zA-Z0-9_]*, single-quoted strings with '' escape, E'...' C-style escapes, numeric literals (int, float, scientific, hex 0x), single-line (--) and nestable block (/* */) comments, whitespace skipping, longest-match operator disambiguation (<> before <, :: before :, || before |), error token collection with source location up to configurable limit (default 50). Include tests for all token types, case insensitivity, string escaping, numeric formats, comment handling, operator disambiguation, error reporting with position, empty input, consecutive strings.

Target files: sqld/src/sql/mod.rs, sqld/src/sql/token.rs, sqld/src/sql/lexer.rs, sqld/src/sql/error.rs, sqld/tests/test_lexer.rs
