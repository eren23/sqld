# Task: B+ tree index with concurrent access

Implement B+ tree index on top of buffer pool pages. Internal nodes: [ptr₀,key₁,ptr₁,...,keyₙ,ptrₙ]. Leaf nodes: [key,tid pairs] + next/prev leaf pointers for range scans. Support: point lookup O(log_B N), range scan O(log_B N + result/B), insert with node splitting (right-split, push median up), delete with lazy marking (merge on VACUUM when <40% full), composite keys (tuple-wise left-to-right compare), descending index (reverse comparator), unique constraint enforcement at insert time. Implement range scan iterator (forward/reverse, inclusive/exclusive bounds). Implement latch crabbing for concurrent access: top-down, release parent latch once child is safe. Include tests: ascending/descending/random key insertion, 100k key balance (height≤3), delete+search, split correctness, unique rejection, composite key prefix/full search, range scans with all bound types, concurrent 8/16-thread insert+search+delete stress test.

Target files: sqld/src/storage/btree/mod.rs, sqld/src/storage/btree/btree.rs, sqld/src/storage/btree/node.rs, sqld/src/storage/btree/iterator.rs, sqld/src/storage/btree/concurrent.rs, sqld/tests/test_btree.rs, sqld/tests/test_btree_scan.rs, sqld/tests/test_btree_concurrent.rs
